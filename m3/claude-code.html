M3 — 学ぶ ｜ まずここから

初めての Claude Code

「概要 → インストール → 初期設定 → 業務ボットを作る」をステップで解説。プログラミング経験ゼロでも大丈夫です。

🙌

プログラミング不要

コードは書きません。「やりたいこと」を日本語で伝えるだけで、Claude Codeが自動で作って動かしてくれます。

1
Overview

Claude Code とは

パソコン上で、ファイルの読み書きやコマンド実行までこなしてくれるAIアシスタントです。「〇〇を自動でやって」と日本語で頼むだけで、必要な処理をAIが組み立てて実行します。

🗂️ ファイルを自動で整理 📊 表計算データを処理 📝 資料を要約・作成 🔁 毎月の作業を自動化

使える場所（インストール先）：

🖥 ターミナル（基本）🧩 VS Code 拡張🧩 JetBrains 拡張🪟 デスクトップアプリ🌐 ブラウザ
2
Install

インストール手順

公式のインストーラを1行コマンドで実行するだけ（Node.jsなどの事前準備は不要です）。お使いのOSを選んでください。

🍎 macOS / 🐧 Linux
ターミナル
$ curl -fsSL https://claude.ai/install.sh | bash
# Homebrew派なら： brew install --cask claude-code

画面イメージ：ターミナルに上記を貼り付けて実行 → 自動でインストールされます。

🪟 Windows（PowerShell）
Windows PowerShell
> irm https://claude.ai/install.ps1 | iex
# WinGet派なら： winget install Anthropic.ClaudeCode

動作環境：macOS / Windows / Linux に対応。ターミナル（コマンド入力画面）を使います。Windowsでは Git を入れておくとより快適です。難しい準備は不要で、コマンドは「コピーして貼り付けて実行」でOKです。

3
Setup

初期設定（初回のみ）

STEP 1

作業フォルダへ移動

整理・作業させたいフォルダで cd します。例：デスクトップなら cd ~/Desktop

STEP 2

Claude Code を起動

コマンド claude と打つだけ。対話画面が立ち上がります。

STEP 3

ログイン

初回はブラウザが開きます。Claudeの有料プラン（Pro/Max/Team等）のアカウントでログイン。

STEP 4

準備完了

以降はログイン不要。/help で使えるコマンド一覧も確認できます。

初回起動の流れ
$ cd ~/Desktop
$ claude
→ ブラウザでログイン画面が開きます…
✓ ログイン完了。何をお手伝いしましょう？

画面イメージ：ログインは初回だけ。次回からは claude ですぐ使えます。

4
Hands-on

かんたんな業務ボットを作ってみる

例：「ダウンロード内の領収書を、月ごとのフォルダに自動で振り分ける」作業を自動化します。コードは一切書きません。

1

やりたいことを日本語で伝える

下のように、普段の言葉で頼むだけ。ファイル名や保存先も書いておくと正確です。

2

Claude Code が確認しながら作る

「移動？コピー？」など必要なことを聞き返し、処理（スクリプト）を自動で用意します。

3

「実行して」で動かす

承認すると実行。「47件を12個の月別フォルダに移動しました」のように結果を報告。

4

来月からは一言でOK

同じ作業は「先月の領収書も整理して」と頼むだけ。あなたの“専用ボット”の完成です。

Claude Code との会話イメージ
あなた） ダウンロードにある今年の領収書（receipt-2026-03-15.pdf など）を、2026-01／2026-02… と月別フォルダに振り分けて。
Claude） 承知しました。移動でよいですか？保存先は Downloads/2026-整理 で作成します。
あなた） それでお願い。実行して。
Claude） ✓ 47件を12個の月別フォルダへ移動しました。
5
No coding

プログラミング不要って本当？

はい。あなたはコードを書きません。やりたいことを言葉で伝え、出てきた結果を確認するだけです。ただし向き・不向きはあるので正直にお伝えします。

✓ 経験ゼロでもできること

  • ファイル整理・名前変更・仕分けの自動化
  • 表計算（Excel/CSV）の集計・整形
  • PDF・資料の要約や抽出
  • 毎月くり返す定型作業の自動化

△ 注意したいこと

  • 思い通りでない時は「ここを直して」と具体的に伝える必要あり
  • 複雑な社内システム連携は技術者の確認が安心
  • エラーが出たら画面の文言をそのまま見せると解決しやすい
