# Cowork セクション — ページ構造・URL設計

Do Lab AI 内に「Cowork」関連ページ群を追加・整理するための、ページ構造とURL設計の検討メモです（Issue #123）。

- **前提**: Cowork は M3「学ぶ」の主要トピック。既存の `m1/ m2/ m3/` セクション体系と、README の URL 設計（フラットなファイル配置）に合わせる。
- **既存ページ**: `m3/cowork.html`（基本的な使い方・#120）、`m3/cowork-features.html`（機能と活用事例・#121）

---

## 1. URL設計（採用：フラット配置）

Cowork のページは **M3「学ぶ」配下にフラットに配置**します（README のURL設計と同じ方式）。

**URL例**：`/m3/cowork.html` ／ `/m3/cowork-features.html`

### 選んだ理由
- README の URL 設計はすべて `/m3/xxx.html` のフラット構成 → **同じ規則で整合**。
- 既に公開済み／PR中の `cowork.html`・`cowork-features.html` の **URL を変えずに済む**。
- 新規ファイルを足すだけでよく、**実装コストが最小**。

> ※ サブフォルダ型（`/m3/cowork/`）や独立セクション型（`/cowork/`）も検討したが、READMEのフラット設計・M3体系と整合する本案を採用。ページが5本以上に増えた段階で、サブフォルダ＋ハブ型への移行を再検討する。

## 5. ページ間の遷移・導線

### 導線ルール
1. **入口**：ダッシュボード →「学ぶ（M3）」→ M3トップのカードから各 Cowork ページへ。
2. **戻り導線**：各 Cowork ページの上部に「← M3 学ぶ トップに戻る」を常設（実装済み）。
3. **相互リンク**：
   - 「基本的な使い方」ページに、より詳しく知りたい人向けに **「機能と活用事例」へのリンク**を置く（実装済み）。
   - 「機能と活用事例」ページに、使い方を知りたい人向けに **「基本的な使い方」へのリンク**を置く（実装済み）。
4. **ヘッダーナビ**：全ページ共通ヘッダーから「学ぶ（M3）／M1／M2」へ移動可能（既存）。
5. 将来ハブを作る場合は、`M3トップ → Coworkハブ → 各ページ` の2階層導線に切り替える。

▼ 図：ページ間の遷移・導線

```
[ダッシュボード /] 
      │  「学ぶ（M3）」
      ▼
[M3トップ /m3/] ──カード──▶ [Cowork 基本的な使い方]  ⇄  [Cowork 機能と活用事例]
                                   │                          │
                          （各ページ上部）            （各ページ上部）
                          「← M3 学ぶ トップに戻る」   「← M3 学ぶ トップに戻る」
```
