1. URL設計（採用：フラット配置）

Cowork のページは M3「学ぶ」配下にフラットに配置します（README のURL設計と同じ方式）。

✅ 採用：フラット配置

URL例：　/m3/cowork.html　／　/m3/cowork-features.html

選んだ理由
  • READMEのURL設計が全て /m3/xxx.html のフラット構成 → 同じ規則で整合。
  • 公開済み／PR中の cowork.html・cowork-features.html の URLを変えずに済む。
  • 新規ファイルを足すだけでよく、実装コストが最小。

※ サブフォルダ型（/m3/cowork/）や独立セクション型（/cowork/）も検討しましたが、READMEのフラット設計・M3体系と整合する本案を採用しました。ページが5本以上に増えた段階で、サブフォルダ＋ハブ型への移行を再検討します。

5. ページ間の遷移・導線

導線ルール

  1. 入口：ダッシュボード →「学ぶ（M3）」→ M3トップのカードから各 Cowork ページへ。
  2. 戻り導線：各 Cowork ページ上部に「← M3 学ぶ トップに戻る」を常設（実装済み）。
  3. 相互リンク：「基本的な使い方」⇄「機能と活用事例」を相互にリンク（実装済み）。
  4. ヘッダーナビ：全ページ共通ヘッダーから「学ぶ（M3）／M1／M2」へ移動可能（既存）。
  5. 将来ハブを作る場合は M3トップ → Coworkハブ → 各ページ の2階層導線に切り替える。

▼ 図：ページ間の遷移・導線

[M3トップ /m3/] ──カード──▶ [Cowork 基本的な使い方]  ⇄  [Cowork 機能と活用事例]
                                   │                          │
                          （各ページ上部）            （各ページ上部）
                          「← M3 学ぶ トップに戻る」   「← M3 学ぶ トップに戻る」

まとめ（結論）

  • 導線：M3トップ起点＋ページ相互リンク＋共通の戻り導線で回遊性を確保。
